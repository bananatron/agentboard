# Agentboard API

Agentboard now exposes a compact HTTP API designed for AI agents and automation. These docs are GitHub Pages friendly — push them to `main` and enable Pages → `docs/` to publish.

## Base URL & Versioning

- **Base URL (local):** `http://127.0.0.1:8080`
- **Base URL (Railway):** `https://<your-domain>.up.railway.app`
- **Version:** `v1` (implicit). Backward-incompatible changes will bump the major version by changing the base path.

## Authentication

Every request must include the API key generated by `agentboard api`:

```
X-API-Key: <AGENTBOARD_API_KEY>
```

`Authorization: Bearer <key>` is also supported. Missing or invalid keys return `401 {"error":"missing_or_invalid_api_key"}`. Never commit your `.env`; the binary writes it with `0600` permissions automatically.

## Common Response Format

All responses are `application/json`. Errors follow:

```json
{
  "error": "not_found"
}
```

Codes used: `400`, `401`, `404`, `500`.

## Endpoints

### `GET /status`

Returns board counts, active agents, enrichment queue, and pending suggestion count.

```json
{
  "columns": {"backlog": 12, "in_progress": 3, "...": 0},
  "total": 25,
  "agents": [
    {"task_id":"abcd...","task_title":"Fix lint","agent":"claude","status":"active","column":"in_progress"}
  ],
  "enrichments": [
    {"task_id":"efgh...","task_title":"Spec RFC","status":"pending","agent":"claude"}
  ],
  "pending_suggestions": 4
}
```

### Tasks

| Method & Path | Description |
| --- | --- |
| `GET /tasks` | List tasks; supports `status`, `assignee`, `search` query params |
| `POST /tasks` | Create task `{ "title": "...", "description": "...", "enrich": bool }` |
| `GET /tasks/{id}` | Fetch single task (includes `blocked_by`) |
| `PATCH /tasks/{id}` | Update fields (`title`, `description`, `status`, `assignee`, `branch`, `pr_url`, `pr_number`, `enrichment_status`, `enrichment_agent`) |
| `DELETE /tasks/{id}` | Delete task |
| `POST /tasks/{id}/claim` | `{ "assignee": "user" }` |
| `POST /tasks/{id}/unclaim` | Clears assignment |
| `POST /tasks/{id}/agent-activity` | `{ "activity": "reviewing diff" }` (200 chars max) |

Task comments and dependencies:

| Method | Path | Body |
| --- | --- | --- |
| `GET` | `/tasks/{id}/comments` | — |
| `POST` | `/tasks/{id}/comments` | `{ "author": "alice", "body": "note" }` |
| `GET` | `/tasks/{id}/dependencies` | — |
| `POST` | `/tasks/{id}/dependencies` | `{ "depends_on": "<task-id>" }` |
| `DELETE` | `/tasks/{id}/dependencies` | `{ "depends_on": "<task-id>" }` |

### Suggestions

| Method & Path | Description |
| --- | --- |
| `GET /suggestions?status=pending&task_id=<optional>` | List suggestions |
| `POST /suggestions` | Global suggestions (proposals) |
| `POST /tasks/{id}/suggestions` | Suggestion tied to a task |
| `GET /suggestions/{id}` | Fetch one suggestion |
| `POST /suggestions/{id}/accept` | Accept |
| `POST /suggestions/{id}/dismiss` | Dismiss |

Request body (`POST /suggestions`):

```json
{
  "task_id": "",                // leave empty for proposals
  "type": "proposal|hint",
  "author": "agent-01",
  "title": "Add test coverage",
  "message": "Details...",
  "reason": "Optional extra context"
}
```

### Filtering & Search

- `GET /tasks?status=in_progress`
- `GET /tasks?assignee=alice`
- `GET /tasks?search=refactor` (case-insensitive substring match against title/description)

### Suggested Workflows

1. **List board → pick task → claim:**
   ```bash
   curl -H "X-API-Key: $KEY" "$BASE/tasks?status=backlog"
   curl -X POST -H "X-API-Key: $KEY" -H "Content-Type: application/json" \
        -d '{"assignee":"agent-7"}' "$BASE/tasks/<id>/claim"
   ```
2. **Report agent heartbeat:** call `/tasks/{id}/agent-activity` with the current stage so humans following along can see progress in the TUI.
3. **Submit proposals:** agents discovering new work can post `type: "proposal"` suggestions; humans accept them to spawn new tasks automatically.

## Hosting with GitHub Pages

1. Commit this `docs/` folder.
2. In the GitHub repo, open **Settings → Pages**.
3. Choose **Source: Deploy from branch**, pick `main`, and set the folder to `/docs`.
4. GitHub Pages will serve these Markdown files via Jekyll (theme configured below).
